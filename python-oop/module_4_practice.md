### 1.1 Описание задачи

Нам поступил заказ на разработку **консольного прототипа личного финансового кабинета**.

Заказчик хочет систему, в которой каждый пользователь может:

- зарегистрироваться в системе (логин + пароль);
- войти в систему по логину и паролю;
- пополнять свой кошелёк (операция «доход»);
- снимать деньги с кошелька (операция «расход»);
- просматривать текущий баланс;
- просматривать историю своих операций;
- выходить из системы.

Данные всех пользователей должны **сохраняться между запусками** программы. 

*В качестве хранилища на данном этапе используется **JSON-файл** (база данных появится позже, архитектура должна это предусматривать).*

Все финансовые операции должны **автоматически записываться в лог-файл** — это требование безопасности.

---

### 1.2 Бизнес-правила

1. Логин пользователя **уникален**. Нельзя зарегистрировать двух пользователей с одинаковым логином.
2. Пароль хранится **только в виде хэша** (открытый пароль нигде не сохраняется).
3. Баланс **не может быть отрицательным**. Если пользователь пытается снять больше, чем есть, операция отклоняется с понятным сообщением.
4. Сумма любой операции должна быть **строго больше нуля**.
5. У каждой операции есть **тип** (`deposit` / `withdraw`), **сумма** и **дата/время**.
6. Лог-файл пишется **автоматически** — пользователь об этом не знает и не управляет им вручную.

### 2.2 Описание классов

#### 📄 `models/user.py` — класс `User`

Представляет **пользователя системы**.

**Атрибуты:**

| Атрибут | Тип | Описание |
|---|---|---|
| `login` | `str` | логин (открытый атрибут) |
| `__password_hash` | `str` | хэш пароля (private) |
| `wallet` | `Wallet` | кошелёк пользователя |

**Методы:**

- `__init__(login, password_hash, wallet=None)` — если `wallet` не передан, создаёт новый пустой `Wallet()`.

- `check_password(password_hash: str) -> bool` — возвращает `True` если хэш совпадает.

- `__str__` — вида `User(login='alice', balance=1500.00 руб.)`

- `__repr__` — вида `User(login='alice')`

- `__bool__` — возвращает `True` если у пользователя есть хоть какие-то деньги на кошельке (т.е. `bool(self.wallet)`).

- `to_dict()` — для сохранения в JSON.

- `from_dict(login, data)` — статический метод. Восстанавливает объект из словаря.

#### 📄 `models/transaction.py` — класс `Transaction`

Представляет **одну финансовую операцию**.

**Атрибуты (задаются в `__init__`):**

| Атрибут | Тип | Описание |
|---|---|---|
| `type` | `str` | тип операции: `"deposit"` или `"withdraw"` |
| `amount` | `float` | сумма операции (всегда > 0) |
| `created_at` | `str` | дата и время создания в формате `"YYYY-MM-DD HH:MM:SS"` |

> Дата и время должны устанавливаться **автоматически** в момент создания объекта с помощью `datetime.now().strftime("%Y-%m-%d %H:%M:%S")`.  
> Параметр `created_at` в `__init__` иметь значение по умолчанию `None` — если передан `None`, устанавливается текущее время; если передана строка (при загрузке из JSON) — используется она.

**Методы:**

- `__str__` — возвращает читаемую строку для пользователя.  
  Пример: `[2025-01-15 14:32:01] deposit: +500.00 руб.` или `[2025-01-15 15:10:44] withdraw: -200.00 руб.`

- `__repr__` — возвращает строку для отладки.  
  Пример: `Transaction(type='deposit', amount=500.0, created_at='2025-01-15 14:32:01')`

- `__bool__` — возвращает `True`, если сумма операции больше нуля (иначе `False`).

- `to_dict()` — возвращает словарь `{"type": ..., "amount": ..., "created_at": ...}` для сохранения в JSON.

- `from_dict(data)` — **статический метод** (`@staticmethod`). Принимает словарь, возвращает объект `Transaction`. Используется при загрузке из JSON.

---

#### 📄 `models/wallet.py` — класс `Wallet`

Представляет **кошелёк пользователя** — хранит список операций и управляет балансом.

**Атрибуты (задаются в `__init__`):**

| Атрибут | Тип | Описание |
|---|---|---|
| `_transactions` | `list[Transaction]` | приватный список операций |

> Список `_transactions` — **protected**-атрибут. Напрямую снаружи не изменяется.

**Свойства (`@property`):**

- `balance` — **только чтение**. Вычисляет и возвращает текущий баланс: сумма всех `deposit` минус сумма всех `withdraw`. Тип — `float`.

- `transactions` — **только чтение**. Возвращает копию списка транзакций (чтобы внешний код не мог изменить внутренний список напрямую).

**Методы:**

- `deposit(amount: float) -> Transaction` — создаёт транзакцию типа `deposit`, добавляет её в список, возвращает объект `Transaction`.  
  Вызывает `ValueError` если `amount <= 0`.

- `withdraw(amount: float) -> Transaction` — создаёт транзакцию типа `withdraw`, добавляет её в список, возвращает объект `Transaction`.  
  Вызывает `ValueError` если `amount <= 0`.  
  Вызывает `ValueError` если `amount > self.balance`.

- `__bool__` — возвращает `True`, если баланс больше нуля.

- `__str__` — возвращает строку вида `Wallet(balance=1500.00 руб., transactions=7)`.

- `to_dict()` — возвращает словарь `{"transactions": [t.to_dict() for t in ...]}`.

- `from_dict(data)` — **статический метод**. Принимает словарь, возвращает объект `Wallet` с восстановленными транзакциями.

#### 📄 `storage/json_storage.py` — классы `JsonStorage` и `UserRepository`

##### Класс `JsonStorage`

Низкоуровневый класс для **чтения и записи JSON-файла**.

**Атрибуты:**

| Атрибут | Тип | Описание |
|---|---|---|
| `_filepath` | `str` | путь к JSON-файлу |

**Методы:**

- `__init__(filepath: str)` — сохраняет путь. Если файл не существует — создаёт пустой файл с содержимым `{}`.

- `_read() -> dict` — читает и возвращает содержимое JSON-файла как словарь. Это **protected**-метод (используется только внутри класса и наследниками).

- `_write(data: dict)` — записывает словарь в JSON-файл. Тоже **protected**.

---

##### Класс `UserRepository`

Наследует `JsonStorage`. Работает с **конкретной структурой данных пользователей**.

Структура JSON-файла:

```json
{
  "alice": {
    "password_hash": "хэш_пароля",
    "wallet": {
      "transactions": [...]
    }
  },
  "bob": {
    "password_hash": "хэш_пароля",
    "wallet": {
      "transactions": []
    }
  }
}
```

**Методы:**

- `__init__(filepath: str)` — вызывает `super().__init__(filepath)`.

- `find_by_login(login: str) -> dict | None` — возвращает словарь с данными пользователя по логину. Если не найден — `None`.

- `save_user(login: str, password_hash: str, wallet: Wallet)` — сохраняет (или обновляет) пользователя в файле. Использует `_read()` и `_write()`.

- `exists(login: str) -> bool` — возвращает `True` если пользователь с таким логином уже есть.

- `all_logins() -> list[str]` — возвращает список всех логинов.

#### 📄 `services/logger.py` — класс `Logger`

**Callable-класс** — объект этого класса можно вызвать как функцию (через `__call__`). Используется как **декоратор** для логирования финансовых операций.

**Атрибуты:**

| Атрибут | Тип | Описание |
|---|---|---|
| `_log_filepath` | `str` | путь к лог-файлу |

**Методы:**

- `__init__(log_filepath: str)` — сохраняет путь к лог-файлу. Если файл не существует — создаёт его.

- `__call__(func)` — принимает функцию, возвращает обёрнутую функцию (wrapper). Обёртка:
  1. вызывает оригинальную функцию;
  2. записывает в лог строку вида:  
     `[2025-01-15 14:32:01] user=alice | operation=deposit | amount=500.0 | result=ok`  
     (или `result=error: <текст ошибки>` если было исключение);
  3. пробрасывает исключение дальше (если оно было).

> **Подсказка:** `Logger` — это объект, который ведёт себя как декоратор. Сначала создаём `logger = Logger("data/operations.log")`, а затем используем `@logger` над методами, которые нужно логировать.

#### 📄 `services/auth_service.py` — класс `AuthService`

Отвечает за **регистрацию и вход** пользователей.

**Атрибуты:**

| Атрибут | Тип | Описание |
|---|---|---|
| `_repository` | `UserRepository` | репозиторий пользователей |

**Методы:**

- `__init__(repository: UserRepository)` — сохраняет репозиторий.

- `hash_password(password: str) -> str` — возвращает SHA-256 хэш пароля в виде строки.  
  Использовать: `import hashlib; hashlib.sha256(password.encode()).hexdigest()`

- `register(login: str, password: str) -> User` — регистрирует нового пользователя.  
  Вызывает `ValueError` если логин уже занят.  
  Возвращает объект `User`.

- `login(login: str, password: str) -> User` — проверяет логин и пароль.  
  Вызывает `ValueError` если логин не найден или пароль неверный.  
  Возвращает объект `User`.

#### 📄 `app/finance_app.py` — класс `FinanceApp`

**Главный класс приложения.** Реализует протокол **контекстного менеджера** (`__enter__` / `__exit__`), что гарантирует сохранение данных даже при ошибке.

**Атрибуты:**

| Атрибут | Тип | Описание |
|---|---|---|
| `_repository` | `UserRepository` | репозиторий |
| `_auth` | `AuthService` | сервис авторизации |
| `_logger` | `Logger` | логгер операций |
| `_current_user` | `User \| None` | текущий авторизованный пользователь |

**Методы:**

- `__init__()` — создаёт все зависимости (репозиторий, сервис авторизации, логгер). Устанавливает `_current_user = None`.

- `__enter__` — возвращает `self`. Можно использовать для инициализации ресурсов (вывести приветственное сообщение, создать папку `data/` если не существует).

- `__exit__(exc_type, exc_val, exc_tb)` — сохраняет данные текущего пользователя если он авторизован. Выводит сообщение о завершении работы. Возвращает `False` (не подавляет исключения).

- `register(login, password)` — делегирует в `_auth.register(...)`, сохраняет созданного пользователя в репозитории.

- `login(login, password)` — делегирует в `_auth.login(...)`, устанавливает `_current_user`.

- `logout()` — сохраняет данные текущего пользователя и устанавливает `_current_user = None`.

- `deposit(amount)` — вызывает `_current_user.wallet.deposit(amount)`. Метод должен быть задекорирован `@logger`.

- `withdraw(amount)` — вызывает `_current_user.wallet.withdraw(amount)`. Метод должен быть задекорирован `@logger`.

- `show_balance()` — выводит текущий баланс.

- `show_history()` — выводит историю транзакций.

### 2.3 Полиморфизм и duck typing

В методе `show_history()` класса `FinanceApp` реализуй поддержку **разных форматов вывода**.

Создай две функции (или класса) в файле `app/finance_app.py`:

- `format_as_text(transaction)` — возвращает строку через `str(transaction)`
- `format_as_table_row(transaction)` — возвращает строку в виде `| 2025-01-15 14:32 | deposit  | +500.00 |`

Метод `show_history(formatter=format_as_text)` должен принимать любой callable, у которого есть сигнатура `formatter(transaction) -> str`. Это и есть **duck typing**: нас не интересует тип форматтера, нам важно только что он умеет делать.

### 2.5 Файл `main.py` — консольное меню

Точка входа в приложение. Здесь находится только консольный интерфейс — никакой бизнес-логики!

Меню должно работать в двух режимах:

**Режим «не авторизован»:**
```
=== Личный финансовый кабинет ===
1. Войти
2. Зарегистрироваться
0. Выйти
```

**Режим «авторизован» (после входа):**
```
=== Привет, alice! Баланс: 1500.00 руб. ===
1. Пополнить кошелёк
2. Снять средства
3. История операций
4. Выйти из аккаунта
0. Завершить программу
```

Весь блок запуска должен быть обёрнут в `with FinanceApp() as app:`.

```python
# main.py — структура (не финальный код, только скелет)

from app.finance_app import FinanceApp

def main():
    with FinanceApp() as app:
        while True:
            # показать меню в зависимости от _current_user
            # обработать выбор пользователя
            # вызвать методы app.*
            pass

if __name__ == "__main__":
    main()
```

### Часть 3. Пример использования `finance_app`

Перед вами **примерная** реализация файла `main.py`.

*Конкретная реализация всегда зависит от разработчика.*

```python
from app.finance_app import FinanceApp, format_as_text, format_as_table_row


def show_guest_menu():
    print("\n=== Личный финансовый кабинет ===")
    print("1. Войти")
    print("2. Зарегистрироваться")
    print("0. Выйти")
    return input("Ваш выбор: ").strip()


def show_user_menu(app: FinanceApp):
    user = app._current_user
    balance = user.wallet.balance
    print(f"\n=== Привет, {user.login}! Баланс: {balance:.2f} руб. ===")
    print("1. Пополнить кошелёк")
    print("2. Снять средства")
    print("3. История операций (текстовый формат)")
    print("4. История операций (табличный формат)")
    print("5. Выйти из аккаунта")
    print("0. Завершить программу")
    return input("Ваш выбор: ").strip()


def main():
    with FinanceApp() as app:
        while True:
            if app._current_user is None:
                choice = show_guest_menu()

                if choice == "1":
                    login = input("Логин: ").strip()
                    password = input("Пароль: ").strip()
                    try:
                        app.login(login, password)
                    except ValueError as e:
                        print(f"Ошибка: {e}")

                elif choice == "2":
                    login = input("Придумайте логин: ").strip()
                    password = input("Придумайте пароль: ").strip()
                    try:
                        app.register(login, password)
                    except ValueError as e:
                        print(f"Ошибка: {e}")

                elif choice == "0":
                    break

                else:
                    print("Неверный выбор.")

            else:
                choice = show_user_menu(app)

                if choice == "1":
                    try:
                        amount = float(input("Сумма пополнения: "))
                        app.deposit(amount)
                    except (ValueError, TypeError) as e:
                        print(f"Ошибка: {e}")

                elif choice == "2":
                    try:
                        amount = float(input("Сумма снятия: "))
                        app.withdraw(amount)
                    except (ValueError, TypeError) as e:
                        print(f"Ошибка: {e}")

                elif choice == "3":
                    app.show_history(formatter=format_as_text)

                elif choice == "4":
                    app.show_history(formatter=format_as_table_row)

                elif choice == "5":
                    app.logout()

                elif choice == "0":
                    break

                else:
                    print("Неверный выбор.")


if __name__ == "__main__":
    main()
```
