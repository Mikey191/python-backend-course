## Что происходит, когда Python выполняет цикл `for`

Цикл `for` — один из первых инструментов, с которыми знакомятся в Python. Вы написали сотни циклов вида `for item in collection:`, и они всегда работали. 

Но что именно Python делает за этой простой строкой — остаётся невидимым до тех пор, пока вы не захотите создать собственный объект, по которому можно итерироваться.

Рассмотрим простейший пример:

```python
numbers = [1, 2, 3]

for n in numbers:
    print(n)
```

Это выглядит как «пройти по каждому элементу списка». Но Python выполняет эту строку через строго определённую последовательность шагов, и понять её — значит понять весь механизм итерации.

Вот что на самом деле делает Python при выполнении `for n in numbers:`:

```python
# Шаг 1: Python вызывает iter() для получения итератора
_iterator = iter(numbers)         # вызывает numbers.__iter__()

# Шаг 2: в цикле Python многократно вызывает next()
n = next(_iterator)               # вызывает _iterator.__next__() → 1
n = next(_iterator)               # вызывает _iterator.__next__() → 2
n = next(_iterator)               # вызывает _iterator.__next__() → 3

# Шаг 3: когда элементы заканчиваются, __next__ выбрасывает StopIteration
next(_iterator)                   # StopIteration — цикл завершается
```

`StopIteration` — это не ошибка в привычном смысле. Это специальное исключение, которое служит сигналом: «элементы закончились, цикл можно завершать». Python перехватывает его автоматически и останавливает цикл без каких-либо сообщений об ошибке.

Функции `iter()` и `next()` вы уже знаете из базового курса Python. Теперь становится понятно, что именно они вызывают под капотом: `iter(obj)` вызывает `obj.__iter__()`, а `next(obj)` вызывает `obj.__next__()`. 

Это и есть протокол итерации — соглашение, которому должен следовать любой объект, желающий работать в цикле `for`.

## Два понятия, которые важно не путать: iterable и iterator

На этом месте почти всегда возникает путаница, поэтому разберём её явно и подробно.

В Python существуют два разных понятия, которые связаны с итерацией, но не являются одним и тем же.

**Итерируемый объект (iterable)** — это любой объект, у которого есть метод `__iter__`. При вызове `__iter__` он возвращает итератор. Список, строка, кортеж, словарь — всё это итерируемые объекты. Их можно передать в `for`, в `list()`, в `sum()`.

**Итератор (iterator)** — это объект, у которого есть метод `__next__`. При каждом вызове `__next__` он возвращает следующее значение. Когда значения заканчиваются — выбрасывает `StopIteration`. Итератор также обязан иметь метод `__iter__`, который возвращает сам себя.

Посмотрим на разницу в поведении:

```python
numbers = [1, 2, 3]           # итерируемый объект (iterable)

# Список сам по себе — не итератор. У него нет __next__.
print(hasattr(numbers, '__iter__'))  # True
print(hasattr(numbers, '__next__'))  # False

# iter() создаёт итератор из итерируемого объекта
iterator = iter(numbers)       # итератор

print(hasattr(iterator, '__iter__'))  # True
print(hasattr(iterator, '__next__'))  # True

# Итератор помнит своё текущее положение
print(next(iterator))   # 1
print(next(iterator))   # 2
print(next(iterator))   # 3
print(next(iterator))   # StopIteration
```

Самое важное различие между ними — это количество возможных проходов:

```python
numbers = [1, 2, 3]

# Список можно итерировать сколько угодно раз
for n in numbers:
    print(n)   # 1 2 3

for n in numbers:
    print(n)   # 1 2 3 — работает снова, список не изменился

# Итератор одноразовый — после исчерпания он «мёртв»
iterator = iter(numbers)

for n in iterator:
    print(n)   # 1 2 3

for n in iterator:
    print(n)   # ничего — итератор исчерпан, цикл не выполняется ни разу
```

Именно это поведение — одноразовость итератора — является частой ловушкой для начинающих. Если вы получили итератор и прошли по нему циклом, повторный цикл по тому же итератору просто ничего не сделает: не вызовет ошибку, не выведет предупреждение, просто выполнит 0 итераций. Запомните это.

Отношение между двумя понятиями можно сформулировать так: каждый итератор является итерируемым объектом, но не каждый итерируемый объект является итератором. Список — итерируемый, но не итератор. Объект, возвращённый `iter(список)` — и то, и другое.

## Первый собственный итератор: класс `Countdown`

Теперь реализуем итератор с нуля. Начнём с максимально простого примера — обратный отсчёт от заданного числа до нуля. Здесь нет никакой бизнес-логики — только чистая механика протокола итерации.

```python
class Countdown:
    """
    Итератор обратного отсчёта от start до 0 включительно.
    Объект одновременно является и итерируемым (есть __iter__),
    и итератором (есть __next__).
    """

    def __init__(self, start):
        self.start = start       # начальное значение, от которого считаем
        self.current = start     # текущее значение — изменяется при каждом вызове __next__

    def __iter__(self):
        # __iter__ должен вернуть объект-итератор.
        # Поскольку этот класс сам является итератором, возвращаем self.
        return self

    def __next__(self):
        if self.current < 0:
            # Значения закончились — сигнализируем об окончании итерации.
            # StopIteration — это не ошибка, это стандартный сигнал завершения.
            raise StopIteration

        # Сохраняем текущее значение, уменьшаем счётчик и возвращаем значение
        value = self.current
        self.current -= 1
        return value
```

Проверим работу:

```python
countdown = Countdown(5)

# Использование в цикле for — Python вызывает __iter__ и __next__ автоматически
for n in countdown:
    print(n, end=" ")
# 5 4 3 2 1 0

# Использование вручную — явный вызов next()
countdown2 = Countdown(3)
print(next(countdown2))   # 3
print(next(countdown2))   # 2
print(next(countdown2))   # 1
print(next(countdown2))   # 0
# next(countdown2)        # StopIteration

# Объект работает везде, где Python ожидает итерируемое
print(list(Countdown(4)))          # [4, 3, 2, 1, 0]
print(sum(Countdown(10)))          # 55 (сумма от 0 до 10)
print(tuple(Countdown(3)))         # (3, 2, 1, 0)
```

Обратите внимание: `Countdown` передаётся напрямую в `list()`, `sum()`, `tuple()` — они все работают через протокол итерации. Любая функция стандартной библиотеки, которая принимает «последовательность», на самом деле принимает любой итерируемый объект.

Важный момент: объект `Countdown` — одноразовый. После того как цикл завершился, `self.current` равен `-1`, и повторный цикл по тому же объекту не выдаст ни одного значения. Это прямое следствие того, что `__iter__` возвращает `self` — оба вызова получат один и тот же объект с уже исчерпанным состоянием.

## Правильная архитектура: разделение итерируемого и итератора

Одноразовость — это проблема в тех случаях, когда один объект нужно итерировать несколько раз независимо друг от друга. Классический пример — два вложенных цикла:

```python
countdown = Countdown(3)

for i in countdown:
    for j in countdown:   # оба цикла используют один и тот же объект
        print(i, j)
```

Если запустить этот код, внутренний цикл исчерпает итератор при первой итерации внешнего, и дальше ничего не произойдёт. Это неправильное поведение.

Решение — разделить итерируемый объект и итератор на два отдельных класса. Итерируемый класс хранит данные. Итератор-класс хранит состояние текущего обхода. Каждый новый вызов `__iter__` создаёт свежий итератор с собственным состоянием.

```python
class NumberRange:
    """
    Итерируемый объект, представляющий диапазон чисел.
    Сам по себе НЕ является итератором — не имеет __next__.
    При каждом вызове __iter__ создаёт новый независимый итератор.
    """

    def __init__(self, start, stop):
        self.start = start
        self.stop = stop

    def __iter__(self):
        # Каждый раз создаём НОВЫЙ объект итератора с начальным состоянием.
        # Благодаря этому несколько циклов по одному NumberRange работают независимо.
        return NumberRangeIterator(self.start, self.stop)

    def __repr__(self):
        return f"NumberRange({self.start}, {self.stop})"


class NumberRangeIterator:
    """
    Итератор для NumberRange. Хранит текущее состояние обхода.
    Создаётся заново при каждом обращении к NumberRange.__iter__.
    """

    def __init__(self, start, stop):
        self.current = start   # текущая позиция в диапазоне
        self.stop = stop       # граница, при достижении которой итерация завершается

    def __iter__(self):
        # Итератор тоже должен быть итерируемым — возвращает себя.
        return self

    def __next__(self):
        if self.current >= self.stop:
            raise StopIteration
        value = self.current
        self.current += 1
        return value
```

Теперь два вложенных цикла работают корректно:

```python
rng = NumberRange(1, 4)

# Внешний цикл вызывает rng.__iter__() → создаёт итератор A
for i in rng:
    # Внутренний цикл вызывает rng.__iter__() → создаёт итератор B (независимый от A!)
    for j in rng:
        print(i, j, end="  ")
    print()

# 1 1  1 2  1 3
# 2 1  2 2  2 3
# 3 1  3 2  3 3

# Можно итерировать сколько угодно раз — каждый раз с нуля
print(list(rng))   # [1, 2, 3]
print(list(rng))   # [1, 2, 3] — работает повторно
```

Выбор между двумя подходами определяется требованиями:

- Если объект должен итерироваться только один раз (файл, сетевой поток, генератор событий) — `__iter__` возвращает `self`, оба метода в одном классе.
- Если объект должен поддерживать независимые множественные обходы — разделяйте на два класса.

## Бесконечные итераторы

Итератор не обязан когда-либо выбрасывать `StopIteration`. Бесконечный итератор продолжает возвращать значения без конца — это легитимный и полезный паттерн.

Типичный пример из веб-разработки — генератор уникальных идентификаторов. Каждая новая запись в базе данных или каждый новый HTTP-запрос должны получить уникальный идентификатор:

```python
class IDGenerator:
    """
    Бесконечный генератор последовательных целочисленных идентификаторов.
    Никогда не выбрасывает StopIteration.
    """

    def __init__(self, start=1):
        self.current = start

    def __iter__(self):
        return self

    def __next__(self):
        # Нет условия StopIteration — итератор бесконечен
        value = self.current
        self.current += 1
        return value
```

С бесконечным итератором нельзя использовать голый цикл `for` или функции `list()`, `sum()` — они никогда не завершатся. Нужно явно ограничивать количество получаемых значений:

```python
id_gen = IDGenerator(start=1000)

# Получаем первые несколько идентификаторов вручную
print(next(id_gen))   # 1000
print(next(id_gen))   # 1001
print(next(id_gen))   # 1002

# Ограничиваем через break
for request_id in id_gen:
    print(f"Обрабатываем запрос #{request_id}")
    if request_id >= 1005:
        break
# Обрабатываем запрос #1003
# Обрабатываем запрос #1004
# Обрабатываем запрос #1005

# Ограничиваем через itertools.islice
from itertools import islice

id_gen2 = IDGenerator(start=1)
first_five = list(islice(id_gen2, 5))
print(first_five)   # [1, 2, 3, 4, 5]
```

`itertools.islice` работает с любым итератором и берёт из него не более N элементов — это стандартный способ получить ограниченное количество значений из бесконечного источника.

## Связь с генераторами

Если вы изучали базовый Python, вы встречали функции-генераторы с ключевым словом `yield`. Теперь, когда протокол итерации объяснён, можно раскрыть, что именно делает `yield`.

Функция-генератор — это синтаксический сахар над `__iter__` и `__next__`. Python автоматически создаёт объект-итератор из такой функции. 

Посмотрим на полностью эквивалентный код — сначала через класс, затем через генератор:

```python
# Вариант 1: класс с __iter__ и __next__
class CountdownClass:
    def __init__(self, start):
        self.current = start

    def __iter__(self):
        return self

    def __next__(self):
        if self.current < 0:
            raise StopIteration
        value = self.current
        self.current -= 1
        return value


# Вариант 2: функция-генератор
def countdown_gen(start):
    while start >= 0:
        yield start      # yield приостанавливает функцию и возвращает значение
        start -= 1       # при следующем вызове __next__ выполнение продолжается отсюда


# Оба варианта производят одинаковый результат
print(list(CountdownClass(5)))   # [5, 4, 3, 2, 1, 0]
print(list(countdown_gen(5)))    # [5, 4, 3, 2, 1, 0]
```

Когда Python встречает `yield` в функции, он превращает эту функцию в фабрику итераторов. Вызов `countdown_gen(5)` не выполняет тело функции — он создаёт объект-генератор, у которого есть `__iter__` и `__next__`. `__next__` запускает тело функции до следующего `yield`, возвращает его значение и приостанавливается. При следующем вызове `__next__` выполнение возобновляется с того места, где остановилось.

Практически: если нужен итератор для разового прохода — генераторная функция с `yield` проще и короче. Если итератор должен поддерживать несколько независимых обходов, иметь дополнительные методы или сложное состояние — класс с `__iter__`/`__next__` предпочтительнее.

## Практический пример: ленивая загрузка файла

Первый реальный пример — обработка больших файлов. В веб-разработке это задача экспорта данных: пользователь запрашивает выгрузку миллиона строк из базы данных в CSV-файл.

Наивный подход — загрузить всё в список и потом отдать. При 1 000 000 строк это означает, что весь файл должен одновременно жить в памяти сервера. Для нагруженного сервера, обрабатывающего сотни запросов одновременно, это неприемлемо.

Итератор решает задачу элегантно: читаем по одной строке, обрабатываем, отдаём — и никогда не держим в памяти больше одной строки одновременно.

```python
class CSVReader:
    """
    Итератор для построчного чтения CSV-файла.
    Не загружает весь файл в память — читает строку за строкой.
    Подходит для файлов любого размера.
    """

    def __init__(self, filepath, delimiter=',', skip_header=True):
        self.filepath = filepath
        self.delimiter = delimiter
        self.skip_header = skip_header
        self._file = None           # файловый объект — открывается в __iter__
        self._header = None         # заголовки столбцов (если skip_header=False)

    def __iter__(self):
        # Открываем файл при начале итерации.
        # Если файл уже был открыт (повторная итерация) — закрываем и открываем снова.
        if self._file:
            self._file.close()
        self._file = open(self.filepath, encoding='utf-8')

        # Читаем и сохраняем строку заголовков
        header_line = self._file.readline().strip()
        self._header = header_line.split(self.delimiter)

        return self

    def __next__(self):
        line = self._file.readline()

        if not line:
            # Достигнут конец файла — закрываем и сигнализируем об окончании
            self._file.close()
            raise StopIteration

        # Разбиваем строку по разделителю и создаём словарь {заголовок: значение}
        values = line.strip().split(self.delimiter)
        return dict(zip(self._header, values))

    def __repr__(self):
        return f"CSVReader(filepath={self.filepath!r}, delimiter={self.delimiter!r})"
```

Создадим тестовый файл и проверим работу:

```python
# Создаём тестовый CSV-файл
import os

test_csv = "users.csv"
with open(test_csv, 'w', encoding='utf-8') as f:
    f.write("id,username,email,role\n")
    f.write("1,alice,alice@example.com,admin\n")
    f.write("2,bob,bob@example.com,user\n")
    f.write("3,carol,carol@example.com,moderator\n")

# Итерируемся по файлу — в памяти одновременно только одна строка
reader = CSVReader("users.csv")

for row in reader:
    print(row)
# {'id': '1', 'username': 'alice', 'email': 'alice@example.com', 'role': 'admin'}
# {'id': '2', 'username': 'bob', 'email': 'bob@example.com', 'role': 'user'}
# {'id': '3', 'username': 'carol', 'email': 'carol@example.com', 'role': 'moderator'}

# Фильтруем прямо при итерации — загружаем только то, что нужно
admins = [row for row in reader if row['role'] == 'admin']
print(admins)
# [{'id': '1', 'username': 'alice', 'email': 'alice@example.com', 'role': 'admin'}]

os.remove(test_csv)
```

Ключевое здесь — строка `return dict(zip(self._header, values))`. Мы превращаем каждую строку CSV в словарь, где ключи — это заголовки столбцов. Это именно тот формат, в котором данные приходят из базы данных или API: словари, а не безымянные кортежи значений.

## Практический пример: пагинация API-ответов

Второй реальный пример — пагинация. Большинство API не возвращают все записи сразу: они разбивают их на страницы (например, по 20 записей). Клиент получает первую страницу, затем вторую и так далее, пока данные не закончатся.

Без итератора код выглядит так:

```python
# Без итератора — явный цикл с управлением страницами
page = 1
all_users = []

while True:
    response = api_client.get(f"/users?page={page}&per_page=20")
    if not response.data:
        break
    all_users.extend(response.data)
    page += 1
```

Это работает, но детали пагинации смешаны с бизнес-логикой. С итератором детали скрываются внутри класса:

```python
class PaginatedResponse:
    """
    Итератор, скрывающий логику постраничной загрузки данных.
    Потребитель просто итерируется по объекту — о страницах ему знать не нужно.
    """

    def __init__(self, endpoint, per_page=20):
        self.endpoint = endpoint    # путь API, например "/api/users"
        self.per_page = per_page    # количество записей на странице
        self._current_page = 1
        self._current_batch = []    # текущая загруженная страница (список записей)
        self._batch_index = 0       # позиция внутри текущей страницы
        self._exhausted = False     # флаг: данные закончились

    def _fetch_page(self, page):
        """
        Имитация HTTP-запроса к API.
        В реальном приложении здесь был бы вызов requests.get() или httpx.get().
        """
        # Имитируем базу данных: 45 пользователей
        all_users = [
            {"id": i, "username": f"user_{i}", "email": f"user_{i}@example.com"}
            for i in range(1, 46)
        ]
        start = (page - 1) * self.per_page
        end = start + self.per_page
        return all_users[start:end]

    def __iter__(self):
        # Сбрасываем состояние при каждом новом обходе
        self._current_page = 1
        self._current_batch = []
        self._batch_index = 0
        self._exhausted = False
        return self

    def __next__(self):
        if self._exhausted:
            raise StopIteration

        # Если текущая страница исчерпана — загружаем следующую
        if self._batch_index >= len(self._current_batch):
            self._current_batch = self._fetch_page(self._current_page)

            if not self._current_batch:
                # Страница пустая — данные закончились
                self._exhausted = True
                raise StopIteration

            print(f"  [загружена страница {self._current_page}, "
                  f"{len(self._current_batch)} записей]")
            self._current_page += 1
            self._batch_index = 0

        # Возвращаем текущую запись и переходим к следующей
        record = self._current_batch[self._batch_index]
        self._batch_index += 1
        return record
```

Посмотрим, как это работает с точки зрения потребителя:

```python
users_api = PaginatedResponse("/api/users", per_page=20)

# Потребителю не нужно знать о страницах — он просто итерируется
count = 0
for user in users_api:
    count += 1

print(f"Всего пользователей: {count}")
# [загружена страница 1, 20 записей]
# [загружена страница 2, 20 записей]
# [загружена страница 3, 5 записей]
# Всего пользователей: 45

# Можно использовать со списочными выражениями и встроенными функциями
active_ids = [u["id"] for u in users_api if u["id"] % 2 == 0]
print(f"Чётные ID: {active_ids[:5]}...")
# [загружена страница 1, 20 записей]
# ...
```

Ключевое наблюдение: тот, кто использует `PaginatedResponse`, не знает и не обязан знать, сколько страниц, как устроены HTTP-запросы, где заканчиваются данные. Он пишет `for user in users_api:` — и получает всех пользователей по одному, независимо от того, сколько их — 45 или 45 000 000. Именно это и есть главная ценность итераторов: они скрывают источник данных и механизм его обхода.

## Краткий обзор `itertools`

Стандартная библиотека Python содержит модуль `itertools`, целиком построенный на протоколе итерации. Каждая функция этого модуля принимает итераторы и возвращает новые итераторы. Понимание `__iter__`/`__next__` делает весь этот модуль прозрачным.

Три функции, которые встречаются в реальных проектах чаще всего:

```python
from itertools import islice, chain, takewhile

# islice — взять первые N элементов из любого итератора
# Особенно полезно для бесконечных итераторов
id_gen = IDGenerator(start=100)
first_five_ids = list(islice(id_gen, 5))
print(first_five_ids)   # [100, 101, 102, 103, 104]

# chain — объединить несколько итерируемых объектов в один поток
admins = [{"id": 1, "role": "admin"}, {"id": 2, "role": "admin"}]
users  = [{"id": 3, "role": "user"},  {"id": 4, "role": "user"}]

for record in chain(admins, users):
    print(record["id"], record["role"])
# 1 admin / 2 admin / 3 user / 4 user

# takewhile — брать элементы, пока выполняется условие
# Останавливается при первом нарушении условия
responses = [200, 200, 201, 404, 200, 500]
successful = list(takewhile(lambda code: code < 400, responses))
print(successful)   # [200, 200, 201] — остановились на 404
```

Все три функции принимают любые итерируемые объекты — в том числе ваши собственные классы с `__iter__` и `__next__`. Правильно реализованный протокол итерации делает объект полноценным участником всей экосистемы Python.

## Неочевидные моменты и типичные ошибки

Несколько вещей, о которых в учебниках обычно не говорят, но которые рано или поздно встречаются на практике.

**Итератор исчерпан — и не предупреждает об этом.** Повторная итерация по исчерпанному итератору не вызывает ошибку — она просто ничего не делает:

```python
it = iter([1, 2, 3])
print(list(it))   # [1, 2, 3]
print(list(it))   # [] — итератор исчерпан, результат пустой список
```

Если ваша функция принимает итерируемое и вдруг перестала возвращать данные — первым делом проверяйте, не передаёте ли вы уже исчерпанный итератор.

**`__len__` не делает объект итерируемым.** Распространённое заблуждение: раз объект «знает свой размер», значит, по нему можно итерироваться. Это неверно:

```python
class DataSet:
    def __init__(self, data):
        self.data = data

    def __len__(self):
        return len(self.data)
    # __iter__ не определён!


ds = DataSet([1, 2, 3])
print(len(ds))   # 3 — работает

for item in ds:  # TypeError: 'DataSet' object is not iterable
    print(item)
```

`for` требует `__iter__`, а не `__len__`. Это разные протоколы.

**`StopIteration` внутри генератора с Python 3.7+.** Если вы пишете генераторную функцию и внутри неё выбрасывается `StopIteration` (например, из вложенного итератора), Python не завершит генератор тихо — он преобразует `StopIteration` в `RuntimeError`. Это изменение было введено в PEP 479:

```python
def bad_generator():
    it = iter([1, 2, 3])
    while True:
        yield next(it)   # когда it исчерпан, next() выбросит StopIteration
        # В Python 3.7+ это станет RuntimeError внутри генератора

# Правильный вариант — явный try/except
def good_generator():
    it = iter([1, 2, 3])
    while True:
        try:
            yield next(it)
        except StopIteration:
            return   # явно завершаем генератор
```

**`__iter__` обязан возвращать объект с `__next__`.** Если `__iter__` вернёт объект без `__next__`, ошибка возникнет не сразу — а только при первой попытке вызвать `next()`:

```python
class Broken:
    def __iter__(self):
        return [1, 2, 3]   # список — не итератор, нет __next__


for item in Broken():
    print(item)
# TypeError: iter() returned non-iterator of type 'list'
```

## Итоги урока

Протокол итерации в Python — это соглашение из двух методов: `__iter__` возвращает итератор, `__next__` возвращает следующее значение или выбрасывает `StopIteration`. Именно через этот протокол работает цикл `for`, функции `list()`, `sum()`, `zip()`, `map()`, `filter()` и весь модуль `itertools`.

Различие между итерируемым объектом и итератором принципиально: первый можно обходить многократно, второй одноразовый. Если объект должен поддерживать независимые одновременные обходы — выносите состояние итерации в отдельный класс-итератор.

Бесконечные итераторы — легитимный паттерн для генераторов идентификаторов, потоков событий и источников данных без заранее известного конца. Работайте с ними через `itertools.islice` или явный `break`.

Генераторные функции с `yield` — это синтаксический сахар над `__iter__`/`__next__`. Используйте их для простых одноразовых итераторов; классы предпочтительны для сложного состояния и многократного обхода.

В следующем уроке мы рассмотрим методы `__enter__` и `__exit__`, которые стоят за конструкцией `with`. Именно через них реализуется гарантированное закрытие файлов, соединений с базой данных и других ресурсов — даже если в процессе работы возникло исключение.
