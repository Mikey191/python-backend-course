# Урок 10. Управление доступом к атрибутам: методы `__getattribute__`, `__getattr__`, `__setattr__`, `__delattr__`

---

На этом этапе курса мы переходим к одному из самых «внутренних» механизмов Python — управлению доступом к атрибутам объектов. 

Это тот уровень, на котором становится понятно, **как именно Python работает с объектами при обращении к их свойствам**.

Здесь важно сразу задать правильный вектор мышления:
мы не просто изучаем магические методы, а разбираемся в том, 

**как интерпретатор Python обрабатывает операции вида**:

```python
obj.x
obj.x = 10
del obj.x
```

Именно под эти операции и существуют методы:

* `__getattribute__`
* `__getattr__`
* `__setattr__`
* `__delattr__`

## Напоминание: как устроены атрибуты в Python

Перед тем как перейти к **магическим методам**, важно восстановить базовую модель.

* **Класс** — это пространство имен.
* **Экземпляр класса** — это отдельный объект со своим пространством имен.

```python
class Point:
    MAX_COORD = 100
    MIN_COORD = 0

    def __init__(self, x, y):
        self.x = x
        self.y = y
```

Здесь:

* `MAX_COORD`, `MIN_COORD` — атрибуты класса
* `x`, `y` — атрибуты экземпляра

Создаем объекты:

```python
pt1 = Point(1, 2)
pt2 = Point(10, 20)
```

Ключевой момент:

* у каждого объекта есть свой `__dict__`
* если атрибут не найден в объекте — Python ищет его в классе

```python
print(pt1.MAX_COORD)  # найдено в классе
```

---

## Критически важный нюанс: запись через self

Рассмотрим типичную ошибку:

```python
class Point:
    MIN_COORD = 0

    def set_bound(self, left):
        self.MIN_COORD = left
```

Что здесь происходит на самом деле?

```python
pt1 = Point()
pt1.set_bound(-100)

print(pt1.__dict__)
```

Результат:

```python
{'MIN_COORD': -100}
```

А в классе значение не изменилось.

Почему?

Потому что:

> **Любое присваивание через `self` создает или изменяет атрибут экземпляра, а не класса.**

Если нужно изменить атрибут класса:

```python
class Point:
    MIN_COORD = 0

    @classmethod
    def set_bound(cls, left):
        cls.MIN_COORD = left
```

## Что происходит при работе с атрибутами (ключевая схема)

**Когда вы пишете**:

```python
value = obj.x
```

Python делает следующее:

```python
1. Вызывает 

obj.__getattribute__('x')
```

```python
2. Если атрибут найден → возвращает значение
```

```python
3. Если НЕ найден → вызывает 

obj.__getattr__('x')
```

---

**Когда вы пишете**:

```python
obj.x = 10
```

```python
→ вызывается 

obj.__setattr__('x', 10)
```

---

**Когда удаляете аттрибут**:

```python
del obj.x
```

```python
→ вызывается

obj.__delattr__('x')
```

> Это фундаментальная схема, которую нужно зафиксировать.

## Метод **getattribute**: полный контроль над чтением

Это **самый важный и самый опасный метод**.

Он вызывается **ВСЕГДА**, при любом обращении к атрибуту.

```python
class Point:
    def __init__(self, x, y):
        self.__x = x
        self.__y = y

    def __getattribute__(self, item):
        print(f"Доступ к атрибуту: {item}")
        return object.__getattribute__(self, item)
```

Проверим:

```python
pt = Point(1, 2)
print(pt._Point__x)
```

Вы увидите сообщение.

---

### Почему обязательно вызывать object.**getattribute**?

Если написать так:

```python
def __getattribute__(self, item):
    return self.item
```

→ произойдет бесконечная рекурсия:

* `self.item` снова вызывает `__getattribute__`
* и так до переполнения стека

**Правильный способ:**

```python
return object.__getattribute__(self, item)
```

---

### Пример: запрет доступа к приватному атрибуту

Если мы хотим запретить конкретный аттрибут:

```python
class Point:
    def __init__(self, x, y):
        self.__x = x
        self.__y = y

    def __getattribute__(self, item):
        if item == "_Point__x":
            raise AttributeError("Доступ запрещен к приватному атрибуту")
        return object.__getattribute__(self, item)
```

Теперь:

```python
pt = Point(1, 2)
print(pt._Point__x)  # ошибка

print(pt._Point__y)  # 2
```

### Немного о `object`

Что такое `object` и почему он вообще здесь появляется? До этого момента он был «невидимым», но теперь без него никуда.

Любой класс, который мы создаём, например `class Point` на самом деле равносилен `class Point(object)`. Это **наследование**, в которое мы будем погружаться позднее.

Но если говорить кратко: все встроенные методы, которые реализованы в классе `object` по умолчанию есть и в нашем созданном классе.

> **`object` — это базовый класс для ВСЕХ классов в Python.**

Когда мы пишем `object.__getattribute__(self, item)`, то мы вызываем стандартную реализацию получения атрибута, которая встроена в Python.

---

### Когда используется **getattribute**

* логирование доступа
* прокси-объекты
* безопасность
* ленивые вычисления (lazy loading)

---

## Метод **getattr**: обработка отсутствующих атрибутов

Этот метод вызывается **ТОЛЬКО если атрибут не найден**.

```python
class Point:
    def __getattr__(self, item):
        print(f"Атрибут {item} не найден")
        return None
```

```python
pt = Point()
print(pt.a)
```

Вывод:

```
Атрибут a не найден
None
```

---

### Важное отличие

| Метод              | Когда вызывается              |
| ------------------ | ----------------------------- |
| `__getattribute__` | всегда                        |
| `__getattr__`      | только если атрибут не найден |

---

### Практическое применение

Возвращать значение по умолчанию:

```python
def __getattr__(self, item):
    return False
```

---

## Метод **setattr**: контроль записи

Вызывается **при любом присваивании**:

```python
obj.x = 10
```

---

### Базовый пример

```python
class Point:
    def __setattr__(self, key, value):
        print(f"Установка {key} = {value}")
        object.__setattr__(self, key, value)
```

---

### Критическая ошибка: **рекурсия**

Если внутри метода `__setattr__` мы вызовем стандартное присваивание (которое мы делали например в `__init__`), то метод `__setattr__` начнет вызывать сам себя.

```python
def __setattr__(self, key, value):
    self.key = value  # ОШИБКА 
```

Почему?

* `self.key = value` снова вызывает `__setattr__`
* бесконечная рекурсия (`RecursionError: maximum recursion depth exceeded`)

---

### Правильные варианты

**1. Через object:**

```python
object.__setattr__(self, key, value)
```

**2. Через **dict**:**

```python
self.__dict__[key] = value
```

---

### Пример: запрет на создание атрибута

```python
class Point:
    def __setattr__(self, key, value):
        if key == "xx":
            raise AttributeError("Нельзя создавать атрибут xx")
        object.__setattr__(self, key, value)
```

---

### Пример: валидация данных

```python
class User:
    def __setattr__(self, key, value):
        if key == "age" and value < 0:
            raise ValueError("Возраст не может быть отрицательным")
        object.__setattr__(self, key, value)
```

---

## Метод **delattr**: контроль удаления

Вызывается при:

```python
del obj.x
```

---

### Пример

```python
class Point:
    def __delattr__(self, item):
        print(f"Удаление атрибута {item}")
        object.__delattr__(self, item)
```

---

### Возможная ошибка

```python
def __delattr__(self, item):
    pass
```

→ атрибут НЕ удаляется

---

### Пример: запрет удаления

```python
class Point:
    def __delattr__(self, item):
        if item == "x":
            raise AttributeError("Нельзя удалять x")
        object.__delattr__(self, item)
```

---

## Полная картина: жизненный цикл атрибута

```
Создание:
obj.x = 10
→ __setattr__

Чтение:
obj.x
→ __getattribute__
    → если нет → __getattr__

Удаление:
del obj.x
→ __delattr__
```

---

## Финальный пример: объединяем всё в одном классе

```python
class Point:
    MAX_COORD = 100
    MIN_COORD = 0

    def __init__(self, x, y):
        # Важно: здесь уже вызывается __setattr__
        self.x = x
        self.y = y

    def __getattribute__(self, item):
        print(f"__getattribute__: доступ к {item}")

        # Запрет доступа к "приватному" x
        if item == "_Point__x":
            raise AttributeError("Доступ запрещен")

        # ОБЯЗАТЕЛЬНО: иначе будет рекурсия
        return object.__getattribute__(self, item)

    def __getattr__(self, item):
        # Срабатывает только если атрибут НЕ найден
        print(f"__getattr__: {item} не существует")
        return False

    def __setattr__(self, key, value):
        print(f"__setattr__: установка {key} = {value}")

        # Пример ограничения
        if key == "z":
            raise AttributeError("Атрибут z запрещен")

        # ВАЖНО: иначе не сохранится
        object.__setattr__(self, key, value)

    def __delattr__(self, item):
        print(f"__delattr__: удаление {item}")

        # Запрет удаления x
        if item == "x":
            raise AttributeError("Нельзя удалять x")

        # ОБЯЗАТЕЛЬНО
        object.__delattr__(self, item)
```

## Когда это действительно нужно

Эти методы используются, когда требуется:

* централизованный контроль над атрибутами
* валидация данных
* создание ORM (например, Django ORM)
* прокси-объекты
* ленивые вычисления
* защита данных

---

## Важное предостережение

Методы `__getattribute__` и `__setattr__`:

* легко ломают поведение класса
* усложняют отладку
* могут вызвать рекурсию

**Использовать их нужно осознанно.**

data = SafeDict()

### **Задача 8**

Реализуйте класс `AccessLogger`, который оборачивает другой объект и логирует доступ к его атрибутам.

Это класс-обёртка (Proxy / Decorator pattern).

Объект этого класса создается так:

```python
user = User("Ivan", 30)
logger = AccessLogger(user)
```

где `User` - это уже реализованный класс (Должен быть в вашей программе).

`AccessLogger` должен принимать любой объект и сохранять его внутри атрибута, например, `_obj`.

Нужно реализовать `__getattribute__`:

* при обращении к атрибуту:

    ```python
    logger.name
    ```

    выводить:

    ```python
    Доступ к атрибуту name
    ```
* затем возвращать значение из **оригинального** объекта

Нужно реализовать `__setattr__`:

* при записи:

    ```python
    logger.age = 40
    ```

    выводить:

    ```python
    Изменение атрибута age = 40
    ```
* изменять значение в **оригинальном** объекте

**Пример использования**:

```python
user = User("Bob", 25)
logger = AccessLogger(user)

# Чтение
print(logger.name)

# Изменение
logger.age = 40
print(user.age)  # изменился в оригинальном объекте

# Добавление нового атрибута
logger.city = "Nalchik"
print(user.city)

# Проверка, что работает через прокси
print(logger.city)
```
