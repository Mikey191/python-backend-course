# Урок 8. CRUD — изменение данных

## Что такое CRUD

До этого урока мы только читали данные — делали `SELECT`. Но база данных должна уметь и изменяться: принимать новые записи, обновлять существующие, удалять ненужные.

**CRUD** — аббревиатура четырёх базовых операций с данными:

| Буква | Операция | SQL-команда |
|---|---|---|
| **C** | Create — создать | `INSERT INTO` |
| **R** | Read — прочитать | `SELECT` |
| **U** | Update — обновить | `UPDATE` |
| **D** | Delete — удалить | `DELETE` |

`SELECT` мы уже освоили. В этом уроке разберём оставшиеся три.

> **Важно перед началом:** команды `INSERT`, `UPDATE` и `DELETE` изменяют данные необратимо. В SQLiteStudio нет кнопки "отменить" для выполненных SQL-команд. Всегда проверяйте условие `WHERE` перед выполнением — особенно в `UPDATE` и `DELETE`.

## INSERT INTO — добавление данных

### Одна строка

Базовый синтаксис вставки одной строки:

```sql
INSERT INTO имя_таблицы (столбец1, столбец2, ...)
VALUES (значение1, значение2, ...);
```

Добавим нового пользователя:

```sql
INSERT INTO users (name, email, city, created_at)
VALUES ('Виктор Зубов', 'viktor@gmail.com', 'Москва', '2024-06-20');
```

Проверяем что запись появилась:

```sql
SELECT * FROM users WHERE email = 'viktor@gmail.com';
```

```
┌────┬──────────────┬──────────────────┬────────┬────────────┐
│ id │ name         │ email            │ city   │ created_at │
├────┼──────────────┼──────────────────┼────────┼────────────┤
│ 16 │ Виктор Зубов │ viktor@gmail.com │ Москва │ 2024-06-20 │
└────┴──────────────┴──────────────────┴────────┴────────────┘
```

`id` получил значение `16` автоматически — `AUTOINCREMENT` продолжает счётчик с последнего существующего значения.

### Порядок столбцов

Порядок столбцов в `INSERT INTO` должен совпадать с порядком значений в `VALUES`. Столбцы не обязательно перечислять в том же порядке что в таблице — главное чтобы значения шли в том же порядке что столбцы:

```sql
-- Порядок изменён — но значения соответствуют своим столбцам
INSERT INTO users (city, name, created_at, email)
VALUES ('Казань', 'Олег Кириллов', '2024-06-21', 'oleg@mail.ru');
```

### Столбцы со значениями по умолчанию

Столбцы с `DEFAULT` или `AUTOINCREMENT` можно не указывать — они заполнятся автоматически:

```sql
-- stock имеет DEFAULT 0 — можно не указывать
INSERT INTO products (name, price, category_id)
VALUES ('Коврик для мыши', 900.00, 2);
```

```sql
SELECT id, name, price, stock FROM products WHERE name = 'Коврик для мыши';
```

```
┌────┬─────────────────┬────────┬───────┐
│ id │ name            │ price  │ stock │
├────┼─────────────────┼────────┼───────┤
│ 23 │ Коврик для мыши │ 900.00 │     0 │
└────┴─────────────────┴────────┴───────┘
```

### Несколько строк за один INSERT

Вместо нескольких отдельных команд — перечисляем наборы значений через запятую:

```sql
INSERT INTO categories (name)
VALUES
    ('Спорт'),
    ('Канцелярия'),
    ('Продукты');
```

```sql
-- Несколько товаров сразу
INSERT INTO products (name, price, stock, category_id)
VALUES
    ('Скакалка',        500.00, 30, 6),
    ('Гантели 5 кг',   1500.00, 15, 6),
    ('Блокнот А5',      200.00, 100, 7);
```

Множественный `INSERT` выполняется быстрее чем несколько одиночных — СУБД делает одну операцию вместо трёх. Это важно при добавлении большого количества строк.

## UPDATE — обновление данных

`UPDATE` изменяет значения в уже существующих строках.

```sql
UPDATE имя_таблицы
SET столбец1 = значение1, столбец2 = значение2, ...
WHERE условие;
```

### Обновление одной строки

Обновим email пользователя:

```sql
UPDATE users
SET email = 'viktor.zubov@gmail.com'
WHERE id = 16;
```

Проверяем:

```sql
SELECT name, email FROM users WHERE id = 16;
```

```
┌──────────────┬────────────────────────┐
│ name         │ email                  │
├──────────────┼────────────────────────┤
│ Виктор Зубов │ viktor.zubov@gmail.com │
└──────────────┴────────────────────────┘
```

### Обновление нескольких столбцов

`SET` принимает несколько пар `столбец = значение` через запятую:

```sql
UPDATE products
SET price = 80000.00, stock = 10
WHERE id = 1;
```

### Обновление нескольких строк

`WHERE` может отобрать несколько строк — `UPDATE` изменит все:

```sql
-- Поднять цену всех товаров из категории "Книги" на 10%
UPDATE products
SET price = ROUND(price * 1.1, 2)
WHERE category_id = 4;
```

Обратите внимание: в `SET` можно использовать текущее значение столбца в выражении — `price * 1.1`. СУБД сначала читает текущее значение, затем записывает новое.

Проверяем:

```sql
SELECT name, price FROM products WHERE category_id = 4;
```

```
┌──────────────────────────┬─────────┐
│ name                     │ price   │
├──────────────────────────┼─────────┤
│ Чистый код. Мартин       │ 1320.00 │
│ Паттерны проектирования  │ 1540.00 │
│ Изучаем Python. Лутц     │ 1980.00 │
│ Грокаем алгоритмы        │ 1210.00 │
└──────────────────────────┴─────────┘
```

### ⚠️ UPDATE без WHERE — главная опасность

```sql
-- Хотели обновить один заказ — забыли WHERE
UPDATE orders SET status = 'cancelled';
```

Этот запрос изменит статус **всех** заказов в таблице — всех восемнадцати. Без `WHERE` `UPDATE` применяется к каждой строке таблицы. Восстановить данные без резервной копии невозможно.

**Правило перед каждым UPDATE:**
1. Сначала напишите `SELECT` с тем же `WHERE` — убедитесь что отбираются именно нужные строки
2. Только потом меняйте `SELECT` на `UPDATE SET ...`

```sql
-- Шаг 1: проверяем что отбирается правильно
SELECT id, status FROM orders WHERE id = 5;

-- Шаг 2: убедились — выполняем UPDATE
UPDATE orders SET status = 'cancelled' WHERE id = 5;
```

## DELETE — удаление данных

`DELETE` удаляет строки из таблицы.

```sql
DELETE FROM имя_таблицы
WHERE условие;
```

### Удаление одной строки

```sql
-- Удалим пользователя которого добавили в этом уроке
DELETE FROM users WHERE id = 16;
```

Проверяем:

```sql
SELECT * FROM users WHERE id = 16;
-- Пустой результат — строка удалена
```

### Удаление нескольких строк

```sql
-- Удалить все отменённые заказы
DELETE FROM orders WHERE status = 'cancelled';
```

### Удаление с подзапросом

`WHERE` в `DELETE` работает так же как в `SELECT` — можно использовать подзапросы:

```sql
-- Удалить все позиции заказов для отменённых заказов
DELETE FROM order_items
WHERE order_id IN (
    SELECT id FROM orders WHERE status = 'cancelled'
);
```

### ⚠️ DELETE без WHERE

```sql
-- Удалит ВСЕ строки из таблицы
DELETE FROM order_items;
```

Как и с `UPDATE` — `DELETE` без `WHERE` применяется ко всем строкам. Применяйте то же правило: сначала `SELECT` с тем же условием, потом `DELETE`.

### Порядок удаления при связанных таблицах

Если таблицы связаны внешними ключами, удалять нужно в правильном порядке — сначала зависимые строки, потом родительские. Нельзя удалить пользователя пока у него есть заказы, и нельзя удалить заказ пока в `order_items` есть его позиции:

```sql
-- Правильный порядок: сначала позиции, потом заказ, потом пользователь
DELETE FROM order_items WHERE order_id = 10;
DELETE FROM orders WHERE id = 10;
-- Теперь можно удалить пользователя если нужно
```

> В SQLite проверка внешних ключей по умолчанию отключена — физически удалить "родителя" при наличии "детей" получится, но данные станут несогласованными. В PostgreSQL такое удаление вызовет ошибку. Хорошая привычка — всегда соблюдать порядок удаления.

## DELETE vs TRUNCATE

`TRUNCATE TABLE` — команда для удаления всех строк таблицы целиком. В SQLite её нет — вместо неё используют `DELETE FROM таблица` без `WHERE`. В PostgreSQL и других СУБД `TRUNCATE` существует и работает быстрее чем `DELETE` без условия.

`DELETE FROM table` без `WHERE` удаляет все строки но счётчик `AUTOINCREMENT` не сбрасывается — следующая вставка получит `id` продолжающий предыдущую последовательность. `TRUNCATE` (там где поддерживается) сбрасывает счётчик в начало. В SQLite `TRUNCATE` отсутствует, используется `DELETE` без `WHERE`.

При необходимости счётчик можно сбросить вручную:

```sql id="lfj6t0"
DELETE FROM sqlite_sequence
WHERE name = 'users';
```

*Однако в реальных приложениях обычно не требуется сбрасывать ID после удаления данных.*

| | `DELETE` без `WHERE` | `TRUNCATE` |
|---|---|---|
| Удаляет все строки | Да | Да |
| Поддержка в SQLite | Да | Нет |
| Поддержка в PostgreSQL | Да | Да |
| Скорость на больших таблицах | Медленнее | Быстрее |
| Сбрасывает счётчик AUTOINCREMENT | Нет | Да |
| Можно добавить `WHERE` | Да | Нет |

В нашем курсе используем `DELETE FROM таблица` без `WHERE` там где нужно очистить таблицу полностью — это стандартный подход для SQLite.

## Типичные ошибки которые ломают данные

### 1. UPDATE / DELETE без WHERE

Уже разобрали — самая опасная ошибка. Правило: всегда пишите `SELECT` с тем же `WHERE` перед изменяющей командой.

### 2. Нарушение уникальности

```sql
-- В таблице users поле email объявлено UNIQUE
INSERT INTO users (name, email, city, created_at)
VALUES ('Тест', 'alexey@gmail.com', 'Москва', '2024-06-20');
-- Ошибка: UNIQUE constraint failed: users.email
```

СУБД защищает уникальность и не даст вставить дубликат. Это хорошо — ошибку лучше получить сразу, чем иметь дублирующиеся данные.

### 3. Нарушение NOT NULL

```sql
-- Поле name объявлено NOT NULL
INSERT INTO users (name, email, city, created_at)
VALUES (NULL, 'test@mail.ru', 'Москва', '2024-06-20');
-- Ошибка: NOT NULL constraint failed: users.name
```

### 4. Несовпадение количества столбцов и значений

```sql
INSERT INTO users (name, email, city, created_at)
VALUES ('Тест', 'test@mail.ru', 'Москва');
-- Ошибка: 4 columns but 3 values
```

Количество столбцов в списке должно точно совпадать с количеством значений в `VALUES`.

### 5. UPDATE с неточным WHERE — изменить не те строки

```sql
-- Хотели обновить товар с id=1, написали city вместо id
UPDATE products SET price = 100.00 WHERE category_id = 1;
-- Изменит все 6 товаров из первой категории
```

Всегда используйте `id` в `WHERE` когда обновляете конкретную строку — это единственный гарантированно уникальный идентификатор.

## Вопросы

1. Что означает аббревиатура CRUD и какая SQL-команда соответствует каждой букве?
2. Нужно ли указывать `id` при вставке строки если столбец объявлен как `AUTOINCREMENT`?
3. Что произойдёт если выполнить `UPDATE orders SET status = 'delivered'` без `WHERE`?
4. Как безопасно проверить что `UPDATE` затронет нужные строки перед его выполнением?
5. Можно ли в `SET` использовать текущее значение обновляемого столбца?
6. Почему при удалении связанных данных важен порядок операций?
7. Чем множественный `INSERT` (несколько строк в одном `VALUES`) лучше нескольких одиночных `INSERT`?
8. Что произойдёт при попытке вставить строку с уже существующим значением в столбце `UNIQUE`?
9. В чём разница между `DELETE FROM table` без `WHERE` и `TRUNCATE TABLE` в контексте счётчика `AUTOINCREMENT`?
10. Можно ли использовать подзапрос в условии `WHERE` команды `DELETE`?
