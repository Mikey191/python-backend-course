# Урок 10. Библиотека sqlite3

## Новый уровень: SQL из Python

В первом модуле мы работали с базой данных напрямую — через SQLiteStudio, вручную, глазами. Это важный этап: вы научились думать реляционно и писать запросы любой сложности.

Но в реальном приложении никто не открывает SQLiteStudio чтобы добавить нового пользователя или обновить статус заказа. Это делает код — Python-скрипт, веб-сервер, фоновая задача. Им нужен способ разговаривать с базой данных.

Во втором модуле мы соединим два инструмента которые уже знаем: Python и SQL. Задача — научиться управлять базой данных из кода: выполнять запросы, получать результаты, правильно структурировать работу с данными.

Начнём с самого базового уровня — библиотека `sqlite3` которая уже встроена в Python. Никаких установок, никаких зависимостей — просто импорт и работа.

---

## Подключение к базе данных

Модуль `sqlite3` входит в стандартную библиотеку Python — он доступен сразу без установки.

```python
import sqlite3
```

### connect()

`sqlite3.connect()` открывает соединение с базой данных. Аргумент — путь к файлу `.db`:

```python
import sqlite3

connection = sqlite3.connect('shop.db')
```

Если файл существует — открывает его. Если нет — создаёт новый пустой файл. Соединение (`connection`) — это объект который представляет открытый канал к базе данных.

Специальное значение `':memory:'` создаёт базу данных прямо в оперативной памяти — без файла на диске. Удобно для тестов и экспериментов: база живёт пока живёт программа.

```python
# База в памяти — исчезнет когда скрипт завершится
connection = sqlite3.connect(':memory:')
```

### cursor()

Соединение само по себе не выполняет запросы. Для этого нужен **курсор** — объект через который отправляются запросы и получаются результаты:

```python
connection = sqlite3.connect('shop.db')
cursor = connection.cursor()
```

Можно представить это так: `connection` — это открытая книга (база данных), `cursor` — это палец которым вы водите по строкам (выполняете запросы).

---

## Выполнение запросов — execute()

`cursor.execute()` принимает строку с SQL-запросом и выполняет его:

```python
import sqlite3

connection = sqlite3.connect('shop.db')
cursor = connection.cursor()

cursor.execute('SELECT * FROM products')
```

После `execute()` результаты хранятся в курсоре — их нужно явно извлечь.

---

## Получение результатов

### fetchone() — одна строка

Возвращает следующую строку из результата. Если строк больше нет — возвращает `None`:

```python
cursor.execute('SELECT name, price FROM products WHERE id = 1')
row = cursor.fetchone()
print(row)
# ('Ноутбук Lenovo IdeaPad', 75000.0)
```

Результат — кортеж. Индексы соответствуют порядку столбцов в запросе: `row[0]` — name, `row[1]` — price.

### fetchall() — все строки

Возвращает список всех строк результата. Каждая строка — кортеж:

```python
cursor.execute('SELECT name, price FROM products WHERE category_id = 4')
rows = cursor.fetchall()

for row in rows:
    print(f'{row[0]}: {row[1]} руб.')

# Чистый код. Мартин: 1320.0 руб.
# Паттерны проектирования: 1540.0 руб.
# Изучаем Python. Лутц: 1980.0 руб.
# Грокаем алгоритмы: 1210.0 руб.
```

> `fetchall()` загружает все строки в память сразу. Для небольших выборок — удобно. Для таблиц с миллионами строк — опасно. В таких случаях используют `fetchmany()`.

### fetchmany() — несколько строк

Возвращает указанное количество строк за один вызов. При следующем вызове — следующую порцию:

```python
cursor.execute('SELECT name FROM products')

batch = cursor.fetchmany(5)   # первые 5
print(batch)
# [('Ноутбук Lenovo IdeaPad',), ('Ноутбук ASUS VivoBook',), ...]

batch = cursor.fetchmany(5)   # следующие 5
print(batch)
# [('Мышь Logitech MX Master',), ...]
```

`fetchmany()` полезен при обработке больших объёмов данных порциями — не загружаем всё в память сразу.

### Итерация по курсору

Курсор можно итерировать напрямую в цикле `for` — это наиболее эффективный способ при больших результатах:

```python
cursor.execute('SELECT name, price FROM products')

for row in cursor:
    print(f'{row[0]}: {row[1]} руб.')
```

Вы можете выполнять несколько запросов через один курсор подряд, но при каждом новом `execute()` предыдущий результат теряется. 

Если нужно работать с результатами первого запроса после выполнения второго — нужно сначала забрать все строки через `fetchall()` в переменную, и только потом выполнять следующий запрос.

## row_factory — словари вместо кортежей

По умолчанию каждая строка — кортеж. Обращаться к данным по индексу неудобно и хрупко: если изменить порядок столбцов в запросе — индексы сломаются.

`sqlite3.Row` делает строки похожими на словари: к данным обращаемся по имени столбца:

```python
connection = sqlite3.connect('shop.db')
connection.row_factory = sqlite3.Row   # устанавливается на соединении

cursor = connection.cursor()
cursor.execute('SELECT name, price, stock FROM products WHERE id = 1')

row = cursor.fetchone()
print(row['name'])    # Ноутбук Lenovo IdeaPad
print(row['price'])   # 75000.0
print(row['stock'])   # 12
```

`row_factory` устанавливается один раз на объекте `connection` — все последующие запросы через это соединение будут возвращать `sqlite3.Row` объекты.

`sqlite3.Row` также поддерживает доступ по индексу и итерацию — он обратно совместим с кортежем. Но именованный доступ предпочтительнее.

Если запрос не нашёл ни одной строки, то `fetchone()`, `fetchall()`, `fetchmany()` вернёт `None`. Это важно проверять в коде — обращение к `row['name']` когда `row` равен `None` вызовет `TypeError`. Правильный паттерн: `row = cursor.fetchone(); if row is not None: ...`

## commit() — когда и зачем

В первом модуле мы видели транзакции в SQL: `BEGIN`, `COMMIT`, `ROLLBACK`. В Python это работает через метод `connection.commit()`.

**Важное правило:** изменяющие операции (`INSERT`, `UPDATE`, `DELETE`) требуют явного `commit()` чтобы зафиксировать изменения в файле базы данных. Без `commit()` изменения видны только в текущем соединении — при закрытии соединения они пропадут.

```python
connection = sqlite3.connect('shop.db')
cursor = connection.cursor()

cursor.execute(
    "INSERT INTO users (name, email, city, created_at) VALUES (?, ?, ?, ?)",
    ('Виктор Зубов', 'viktor@gmail.com', 'Москва', '2024-06-25')
)

connection.commit()   # зафиксировать изменения в файле
```

`SELECT`-запросы `commit()` не требуют — они только читают данные.

> Знак вопроса `?` в запросе — это параметризованный запрос. Мы подробно разберём их в следующем уроке. Пока просто запомните что данные передаются отдельно от строки запроса.

---

## Закрытие соединения — close()

После завершения работы соединение нужно закрыть:

```python
connection = sqlite3.connect('shop.db')
cursor = connection.cursor()

cursor.execute('SELECT COUNT(*) FROM products')
print(cursor.fetchone())

cursor.close()        # закрыть курсор
connection.close()    # закрыть соединение
```

Незакрытые соединения держат файл базы данных заблокированным и потребляют ресурсы. Но ручное закрытие — ненадёжный подход: если между `connect()` и `close()` возникнет исключение, `close()` никогда не выполнится.

---

## with — контекстный менеджер

Правильный способ работы с соединением — через `with`. Это контекстный менеджер: при выходе из блока `with` соединение закрывается автоматически — даже если произошло исключение.

```python
import sqlite3

with sqlite3.connect('shop.db') as connection:
    cursor = connection.cursor()
    cursor.execute('SELECT name, price FROM products LIMIT 5')
    rows = cursor.fetchall()
    for row in rows:
        print(row)
# Соединение закрыто автоматически
```

У `with sqlite3.connect()` есть особенность: при успешном завершении блока он автоматически вызывает `commit()`. При исключении — `rollback()`. Это удобно для изменяющих операций.

Полный пример с `row_factory` и `with`:

```python
import sqlite3

with sqlite3.connect('shop.db') as connection:
    connection.row_factory = sqlite3.Row
    cursor = connection.cursor()

    cursor.execute('''
        SELECT p.name, p.price, c.name AS category
        FROM products AS p
        INNER JOIN categories AS c ON p.category_id = c.id
        WHERE p.price > 10000
        ORDER BY p.price DESC
    ''')

    for row in cursor:
        print(f'{row["name"]} — {row["price"]} руб. ({row["category"]})')
```

```
Ноутбук Lenovo IdeaPad — 75000.0 руб. (Электроника)
Ноутбук ASUS VivoBook — 68000.0 руб. (Электроника)
Планшет Apple iPad — 55000.0 руб. (Электроника)
...
```

## Обработка исключений

Работа с файлом базы данных может завершиться ошибкой: файл не найден, нарушено ограничение, синтаксическая ошибка в запросе. Все ошибки `sqlite3` наследуются от `sqlite3.Error`.

```python
import sqlite3

try:
    with sqlite3.connect('shop.db') as connection:
        connection.row_factory = sqlite3.Row
        cursor = connection.cursor()

        cursor.execute('''
            INSERT INTO users (name, email, city, created_at)
            VALUES (?, ?, ?, ?)
        ''', ('Тест', 'alexey@gmail.com', 'Москва', '2024-06-25'))

except sqlite3.IntegrityError as e:
    print(f'Ошибка целостности данных: {e}')
    # Ошибка целостности данных: UNIQUE constraint failed: users.email

except sqlite3.OperationalError as e:
    print(f'Ошибка операции: {e}')
    # Например: no such table: users

except sqlite3.Error as e:
    print(f'Ошибка базы данных: {e}')
```

Основные исключения `sqlite3`:

| Исключение | Когда возникает |
|---|---|
| `sqlite3.IntegrityError` | Нарушение ограничения: `UNIQUE`, `NOT NULL`, `FOREIGN KEY` |
| `sqlite3.OperationalError` | Таблица не существует, файл недоступен, синтаксис SQL |
| `sqlite3.ProgrammingError` | Ошибка в коде: неверное число параметров, закрытый курсор |
| `sqlite3.Error` | Базовый класс для всех ошибок `sqlite3` |

Ловить `sqlite3.IntegrityError` отдельно от `sqlite3.OperationalError` полезно — они требуют разной реакции. Нарушение `UNIQUE` — это ожидаемая ситуация в логике приложения. Несуществующая таблица — это баг в коде.

## Полная картина: от connect() до close()

```python
import sqlite3

# 1. Открываем соединение через контекстный менеджер
with sqlite3.connect('shop.db') as connection:

    # 2. Настраиваем row_factory для удобного доступа к данным
    connection.row_factory = sqlite3.Row

    # 3. Создаём курсор
    cursor = connection.cursor()

    try:
        # 4. Выполняем запрос
        cursor.execute('SELECT name, city FROM users WHERE city = ?', ('Москва',))

        # 5. Получаем результаты
        rows = cursor.fetchall()

        # 6. Работаем с данными
        for row in rows:
            print(f'{row["name"]} — {row["city"]}')

    except sqlite3.Error as e:
        print(f'Ошибка: {e}')

# 7. Соединение закрыто автоматически
```
